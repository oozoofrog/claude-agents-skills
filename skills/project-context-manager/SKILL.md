# Project Context Manager

프로젝트의 개발 연속성을 유지하기 위한 중앙 집중식 `project.json` 파일을 관리합니다.

## 핵심 책임

`project.json`을 **단일 진실 소스(Single Source of Truth)**로 유지하여:
- 세션 간 컨텍스트 보존
- 기능/이슈/설계결정 추적
- 개발 진행 상황 가시화

## 언제 적용되는가

- 새 프로젝트 시작 시
- 새 세션에서 이전 작업 이어갈 때
- 기능 구현 완료/이슈 해결 시
- 프로젝트 현황 파악 요청 시
- 버그/기술 부채 발견 시

## 운영 가이드라인

### 새 project.json 생성 시
1. 기존 코드베이스 구조 분석
2. 사용자에게 프로젝트 목표, 기술 스택, 현재 상태 질문
3. 모든 관련 섹션 초기화
4. 타임스탬프 및 버전 정보 설정

### project.json 업데이트 시
1. 항상 현재 파일을 먼저 읽어 상태 파악
2. 관련 섹션만 타겟 업데이트
3. 기록 유지 (삭제 대신 completed/resolved 표시)
4. `lastUpdated` 타임스탬프 갱신
5. 중요 변경 시 changelog 추가

### 새 세션 시작 시
1. project.json 분석
2. 현재 상태 요약 제공
3. 대기 중인 작업, 활성 기능, 차단 요소 강조
4. 마지막 세션의 `nextSteps` 제시
5. 추적되지 않은 업데이트 여부 확인

## 품질 보증

- JSON 구조 유효성 검증
- 모든 ID 고유성 및 일관된 형식 유지
- 관련 항목 간 참조 무결성 유지
- 날짜 형식: ISO 8601 (YYYY-MM-DD)

## 중요 규칙

1. 명시적 확인 없이 기존 데이터 덮어쓰지 않음
2. 구조 변경 시 백업 제안
3. 오래된 정보 정리를 위한 주기적 검토 제안
4. 데이터 불일치/충돌 발견 시 플래그
5. 가독성 있는 형식 유지 (적절한 들여쓰기)
