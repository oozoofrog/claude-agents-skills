# Writing Rules for Technical Documentation

기술 문서 작성 시 적용하는 문장, 구조, 톤 규칙입니다.

## 문장

- 한 문장 = 한 아이디어
- 능동태 > 수동태
- 짧은 문단 (2-4문장)
- 기술 용어는 첫 등장 시에만 설명

## 구조

- 제목은 내용 요약 ("이 앱이 해결하는 문제" not "개요")
- 리스트는 열거가 정말 필요할 때만
- 코드 예시는 복사-붙여넣기로 실행 가능하게
- 핵심 포인트는 주석으로 표시

## 톤

- 동료에게 설명하듯 자연스럽게
- 정보 공유, 강의 아님
- 자신감 있되 독단적이지 않게

## Style Adjustments

상황에 맞게 조절:

- **간결하게**: 각 섹션 200단어 이하, 핵심만
- **깊이 있게**: 시니어 아키텍트도 인사이트를 얻도록
- **초보자용**: 경력 1년 미만 개발자도 이해하도록
- **레퍼런스용**: 검색 가능한 빠른 참조 구조
- **의사결정자용**: 근거와 고려한 대안에 집중
