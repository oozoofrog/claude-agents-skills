# Weekly Report Generator

Git 커밋 히스토리를 분석하여 주간보고 형식의 문서를 생성합니다.

## 분석 대상

### 기본 사용자
- Git config에서 자동 감지 (`git config user.name`)
- 사용자가 다른 이름을 지정하면 해당 이름으로 검색

### 검색 범위 (주간보고 기준)

| 섹션 | 기간 | 설명 |
|------|------|------|
| **[한일]** | 지난 주 월요일 ~ 일요일 | 완료된 작업 |
| **[할일]** | 이번 주 월요일 ~ 오늘 | 진행 중인 작업 |

## 기본 영역 분류

1. **기능 개발**: 신규 기능, 버그/크래시 수정
2. **리팩토링**: 코드 정리, 문서화, 구조 개선
3. **유지보수**: 경고 제거, deprecated API 수정, 테스트 개선
4. **개발환경 개선**: 개발 도구 설정, CI/CD, 빌드 설정

**참고**: 영역은 프로젝트에 맞게 커스터마이즈 가능합니다.

## 같은 티켓 통합 규칙 (필수)

같은 이슈 트래커 티켓 번호를 가진 여러 커밋은 **반드시 하나의 항목으로 통합**:
1. 같은 티켓 번호 커밋들을 그룹화
2. 작업의 핵심 목적을 파악하여 요약 제목 생성
3. 상세 구현 내용은 생략하고 핵심만 표시

## 제외 항목
- 이전 차수에서 이미 보고된 항목
- 사소한 설정 변경
- 머지 커밋

## 핵심 규칙 요약

1. **이슈 트래커 연동** (JIRA MCP 있으면 사용): 티켓이 있으면 상세 정보 조회
2. **분류 우선순위**: 이슈 트래커 정보 > 커밋 메시지 키워드
3. **작업 유형으로 세부 분류**: Fix→개발, Refactor→리팩토링, deprecated→유지보수
4. **같은 티켓 통합 (필수)**
5. **제외 항목 준수**
