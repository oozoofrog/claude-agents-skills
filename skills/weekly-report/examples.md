# Examples for Weekly Report

출력 템플릿과 실제 예시입니다.

## 출력 템플릿

```
[한일]
{영역1}
{차수}
- {작업내용1}({상태})
- {작업내용2}({상태})

{영역2}
{차수}
- {작업내용3}({상태})

[할일]
{영역1}
{차수}
- {예정작업1}
- {예정작업2}

{영역2}
{차수}
- {예정작업3}
```

## 예시 출력

**서로 다른 차수의 티켓들이 올바르게 분리된 예시:**

```
[한일]
기능 개발
v1.5
- [PROJECT-101] 사용자 프로필 수정 기능 추가(완료)
- [PROJECT-102] 로그인 화면 크래시 수정(완료)
- [PROJECT-105] 검색 결과 페이지네이션 버그 수정(완료)

리팩토링
v1.5
- [PROJECT-110] API 클라이언트 코드 리팩토링(진행)
- Sample App 프로젝트 재구성(완료)

유지보수
v1.5
- 간단한 경고 제거(완료)

[할일]
리팩토링
v1.6
- [PROJECT-115] 데이터 모델 문서화

유지보수
v1.6
- [PROJECT-112] deprecated API 대응
- [PROJECT-118] 테스트 케이스 가독성 개선

개발환경 개선
v1.6
- CI/CD 파이프라인 개선
- 개발 도구 설정 가이드 추가
```

**차수 결정 과정:**
- PROJECT-101~110: JIRA fixVersions가 `v1.5.0` → `v1.5`
- PROJECT-112~118: JIRA fixVersions가 `v1.6.0` → `v1.6`

## 사용 예시

```
# 기본 사용 (git config user.name 사용, 자동 날짜 계산)
"주간보고 작성해줘"
→ [한일]: 지난 주 월~일
→ [할일]: 이번 주 월~오늘

# 다른 사용자
"kim.developer의 주간보고 만들어줘"

# 차수 지정
"Sprint 15 주간보고 생성해줘"

# 할일에 예정 작업 추가
"주간보고 작성하고, 할일에 '코드 리뷰' 추가해줘"
```

## 주의사항

- git 저장소가 아닌 디렉토리에서는 동작하지 않음
- 원격 저장소 접근 권한 필요
- 커밋 메시지 형식이 불규칙하면 분류 정확도 저하
- **[한일]**: 지난 주 월~일 커밋 (완료된 작업)
- **[할일]**: 이번 주 월~오늘 커밋 + 사용자 입력 예정 작업
- 월요일에 실행하면 [할일]이 오늘 하루만 포함됨
