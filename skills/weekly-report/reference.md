# Reference Tables for Weekly Report

영역 분류를 위한 키워드 참조 테이블입니다.

## 작업 영역 분류 키워드

| 키워드 | 영역 |
|--------|------|
| 크래시, Crash, 버그, Bug, Fix, 기능, 신규, 추가, Feature, 구현, 수정, 해결 | 기능 개발 |
| Refactor, 리팩토링, Sample, 샘플, 문서화, Doc, Documentation, 구조 개선, 정리, 개선 | 리팩토링 |
| Warning, 워닝, 경고, deprecated, Deprecated, Test, 테스트, 퍼포먼스, Performance, 안정성 | 유지보수 |
| CI, CD, 빌드, Build, 개발환경, DevEnv, 설정 | 개발환경 개선 |

## 작업 유형 → 영역 매핑

| 작업 유형 키워드 | 분류 영역 |
|----------------|----------|
| `크래시`, `Crash`, `버그`, `Bug`, `Fix`, `수정`, `해결` | 기능 개발 |
| `신규`, `추가`, `기능`, `Feature`, `구현` | 기능 개발 |
| `Refactor`, `리팩토링`, `개선`, `정리`, `문서화` | 리팩토링 |
| `Warning`, `deprecated`, `테스트`, `경고`, `워닝` | 유지보수 |

## 같은 티켓 통합 예시

| 원본 커밋들 | 통합 결과 |
|------------|----------|
| `[ISSUE-123] API deprecated 수정 (1/3)` | `[ISSUE-123] deprecated API 전환` |
| `[ISSUE-123] API deprecated 수정 (2/3)` | (위와 통합) |
| `[ISSUE-123] API deprecated 수정 (3/3)` | (위와 통합) |
| `[ISSUE-456] 결과 표시 개선` | `[ISSUE-456] 결과 표시 로직 개선` |
| `[ISSUE-456] 콜백 문서화` | (위와 통합) |

## 이슈 티켓 패턴 (프로젝트별 커스터마이즈)

- `[PROJECT-\d+]` - 프로젝트 티켓
- `[ISSUE-\d+]` - 이슈 티켓
- `#\d+` - GitHub 이슈 번호
- `[NO-JIRA]`, `[NO-ISSUE]` - 티켓 없음 (제거하고 작업 내용만 표시)

## 영역 분류 원칙

1. **하나의 커밋 = 하나의 영역**: 여러 영역에 걸치면 주요 작업 기준으로 분류
2. **작업 유형 우선**: 같은 모듈이라도 Fix는 개발, Refactor는 리팩토링
3. **키워드 순서**: 커밋 메시지 앞부분 키워드 우선 적용
